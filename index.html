🧠 Алгоритм самообучения

Как работает автоопределение столбцов

При обработке Excel-файлов приложение анализирует содержимое ячеек в каждом столбце:

  • ЮЛ — если в ячейках обнаружены названия организаций (ООО, АО, ПАО и др.)
  • ФИО — если в ячейках обнаружены фамилии и инициалы (Иванов А.А. и др.)
  • Логин — если в ячейках обнаружены логины (BALANDINAIY и др.)

При достижении порога совпадений (>30% ячеек) весь столбец автоматически типизируется и маскируется.

💡 Совет: Добавьте свои названия столбцов выше для более точного маскирования специфичных данных вашей организации.

Настройки маскирования

Параметры маски

🔤 Символ маски

Символ, которым заменяются обнаруженные данные. По умолчанию используется *.

🔄 Режим маскирования

Определяет как именно заменяются данные:

  • Сохранять длину — каждый символ заменяется символом маски
  • Полная замена — данные заменяются на "[ЗАМАСКИРОВАНО]"
  • Частичное скрытие — последние 4 символа видны

🎯 Маскировать ТОЛЬКО указанные столбцы

Когда включено — маскируются только столбцы, названия которых добавлены в списки "Коммерческая тайна" и "Персональные данные" (вкладка "Колонки"). Все остальные столбцы остаются без изменений.

Пользовательские паттерны

📐 Добавление своих паттернов

Вы можете добавить собственные правила поиска и маскирования данных используя регулярные выражения (regex).

  • Укажите название паттерна для удобства идентификации
  • Введите регулярное выражение (regex) для поиска данных
  • Используйте кнопку Добавить паттерн для создания нового правила

Пример regex: \b\d{10}\b

🔢 Маскирование всех цифр в столбцах

Столбцы, где все цифры заменяются на маску

Укажите названия столбцов, в которых все цифры (0-9) будут заменены на символ маски. Полезно для скрытия числовых идентификаторов, кодов, артикулов.

  • Проверяется по подстроке (регистр неважен)
  • Работает для Excel (XLSX) файлов
  • Приоритет: исключения > маскирование цифр > мягкое/жёсткое маскирование

⚠️ В указанных столбцах все цифры будут заменены на символ маски

🔓 Деобфускация (реверс-восстановление)

Хеширование с возможностью обратной замены

Режим хеширования при маскировании

При включении вместо звёздочек подставляются хеши. Одинаковые значения получают одинаковый хеш.

✓ Активно: Маскирование заменит данные на хеши формата #ORG_xxxxxxxx (организации), #FIO_xxxxxxxx (ФИО), #USR_xxxxxxxx (логины). Включая русские и английские данные.

📐 Поля для хеширования

Обязательные поля

Следующие типы данных автоматически заменяются на хеши при включённом режиме:

  • Название организации (русские и английские) — хеш формата #ORG_xxxxxxxx
  • ФИО (русские и английские) — хеш формата #FIO_xxxxxxxx
  • Учетная запись / Логин — хеш формата #USR_xxxxxxxx
  • Email — хеш формата #EML_xxxxxxxx
  • Телефон — хеш формата #PHN_xxxxxxxx
  • URL-адреса — хеш формата #URL_xxxxxxxx

🔄 Режим «Накопление хешей»
